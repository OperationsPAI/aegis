# FORGE Ablation Driver

One-shot harness for the NeurIPS 2026 paper's ablation table
(`tables/ablations_main.tex`) and Appendix §`sec:appendix_ablations`.

## What it does

Runs the v3 reasoning pipeline against every case in a dataset under six
configurations and emits a single JSON + LaTeX-snippet pair that the
paper repo can ingest.

Configurations:

| Name             | Hook                                                    | Modes                |
|------------------|---------------------------------------------------------|----------------------|
| `baseline`       | Default 4-gate pipeline + builtin rules                 | `fault_free`, `real` |
| `skip_topology`  | Replace rule set R with admit-all (every kind tuple)    | `fault_free`, `real` |
| `skip_screen`    | Drop DriftGate from the gate stack                      | `fault_free`, `real` |
| `slo_x0.5`       | Scale `get_adaptive_threshold` output by 0.5 (tighter)  | `real` only          |
| `slo_x1.5`       | Scale by 1.5 (looser)                                   | `real` only          |
| `slo_x2.0`       | Scale by 2.0 (looser)                                   | `real` only          |

Modes:

* **real** — full pipeline against actual abnormal traces with the
  resolved real injection node. Yields `attributed_rate` and per-case
  `n_paths`.
* **fault_free** — equivalent to `sham_injection_fp.py --mode v2`:
  splits `normal_traces` in half (first = baseline, second = pretend-
  abnormal) AND swaps the injection node for a random non-GT node of
  matching `PlaceKind`. Any `attributed` label is a joint FP under
  this configuration's gate stack × rule set.

## Usage

```bash
# Smoke (20 cases, all 6 configs, ~3 min on 12 workers)
uv run python bin/paper_artifacts/ablations_table.py \
    --dataset /home/ddq/AoyangSpace/dataset/rca \
    --workers 12 --max-cases 20 \
    --out output/ablations/smoke.json

# Full run (500 cases, all 6 configs, ~30-90 min on 12 workers)
uv run python bin/paper_artifacts/ablations_table.py \
    --dataset /home/ddq/AoyangSpace/dataset/rca \
    --workers 12 \
    --out output/ablations/ablations_table.json

# Subset of configs
uv run python bin/paper_artifacts/ablations_table.py \
    --dataset /home/ddq/AoyangSpace/dataset/rca \
    --workers 12 \
    --configs baseline slo_x0.5 slo_x1.5 \
    --out output/ablations/slo_only.json
```

### Caveats

* **`skip_topology` is capped at `max_hops=5`** (vs the default 15).
  With an admit-all rule set the corridor explorer cannot prune;
  candidate-path enumeration is exponential in `max_hops` and at the
  default budget the pipeline does not terminate within minutes per
  case. The cap is reported on each summary as `max_hops`. This
  computational explosion is itself part of the ablation finding:
  the rule set is what makes the pipeline tractable.
* `slo_x*` configs **monkey-patch** `parquet_loader.get_adaptive_threshold`
  process-globally. Each `(config, mode)` invocation creates a fresh
  ProcessPoolExecutor, so workers do not leak patched state across
  configs.
* `fault_free` mode requires `normal_traces.parquet` with at least
  two timestamps; cases that don't satisfy this report
  `error == "no-baseline-data"` and are excluded from the rate
  denominator.

## Output format

Two artifacts under `--out`:

1. `<out>.json` — structured summary, schema version `"1"`:

   ```json
   {
     "version": "1",
     "dataset": "...",
     "n_cases": 500,
     "configs": {
       "baseline": {
         "fault_free": {
           "n_cases": 500, "n_classified": ...,
           "labels": {"attributed": 18, "unexplained_impact": ..., ...},
           "attributed": 18, "attributed_rate": 0.036,
           "joint_fp_rate": 0.036, "path_count_mean": ...,
           "max_hops": 15, "elapsed_seconds": ...
         },
         "real": { ... }
       },
       "skip_topology": { ... },
       ...
     },
     "git_sha": "...",
     "timestamp": "...",
     "args": { ... }
   }
   ```

2. `<stem>_cells.tex` — `\newcommand` definitions for paper repo's
   `scripts/extract_experiment_data.py`. Naming pattern:

   ```latex
   \newcommand{\ablbaselineAttrRate}{99.8\%}
   \newcommand{\ablbaselinePathCount}{510.20}
   \newcommand{\ablbaselineJointFP}{3.6\%}
   ...
   ```

## How to feed back to the paper

The paper repo at `/home/ddq/AoyangSpace/rcabench_paper` reads
experimental numbers from `generated/experiment_vars.tex`, which is
auto-generated by `scripts/extract_experiment_data.py`. To wire the
ablations in:

1. Copy or symlink `output/ablations/ablations_table.json` to a path
   the paper repo's extraction script can read.
2. Extend `scripts/extract_experiment_data.py` to read the JSON and
   emit the corresponding `\newcommand` lines. (The
   `<stem>_cells.tex` file written by this driver is a drop-in
   reference for the macro names used.)
3. Replace the `\needsfill{TBD}` cells in
   `tables/ablations_main.tex` with the new macros.

## Reproducibility

* Sham-target selection in `fault_free` mode uses a deterministic
  seed derived from `case_name` (md5 prefix), so reruns produce the
  same target per case.
* The driver records `git_sha` and `timestamp` in the JSON.

## Failure modes / debugging

* `n_errors > 0` in any summary: open the JSON and read `errors` by
  category (e.g. `resolve`, `no-baseline-data`, `RuntimeError`). The
  first 5 raw rows are also stashed under `sample_rows` for that
  configuration.
* `attributed_rate` is computed over `n_classified` (cases with a
  non-null label), not `n_cases`; an erroring run does not inflate or
  deflate the rate.
* For `skip_topology`, if cases time out or run out of memory at
  `max_hops=5`, lower it further with `--max-hops 3`.
