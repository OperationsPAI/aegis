{{- if and (not .Values.chaosAuth.secretName) (not .Values.chaosAuth.literal) }}

WARNING: chaosAuth.requireBearer is true but neither chaosAuth.secretName
nor chaosAuth.literal is configured. The chaos pod will reject every
/v1beta request with 401. Either:

  - set chaosAuth.literal to a shared secret (dev/CI), or
  - set chaosAuth.secretName to an existing Secret whose
    chaosAuth.secretKey holds the bearer token, or
  - set chaosAuth.requireBearer=false (NOT recommended in production).

{{- end }}
{{- end }}

{{- if not .Values.serviceAccount.secretName }}

NOTICE: serviceAccount.secretName is unset. The chaos pod will fall back
to the DEPRECATED webhookBearer / chaosAuth.* static-bearer path for
outbound webhook auth and log an ERROR at boot. Provision a
rcabench-chaos-sa Secret holding a service-account JWT minted via
  POST /v1/service-accounts/chaos-service/issue
and set chaos.serviceAccount.secretName accordingly. The chaosAuth.*
block remains supported for the inbound /v1beta direction during the
cutover but is on a deprecation path.

{{- end }}
